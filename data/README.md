# GeoIPデータ管理

このフォルダにはGeoIP制限機能で使用するIPアドレスデータが格納されています。

## 📁 ファイル構成

### `japan-ip-ranges.json`
日本国内のIPアドレス範囲をプロバイダー別に管理します。

```json
{
  "name": "日本IPアドレス範囲",
  "description": "JPNIC割り当ての正確な日本IPアドレス範囲",
  "version": "1.0.0",
  "lastUpdated": "2026-03-27",
  "ranges": [
    {
      "provider": "NTT Communications",
      "ranges": ["1.0.16.0/20", "1.0.64.0/18", ...]
    },
    {
      "provider": "KDDI", 
      "ranges": ["2.28.0.0/15", "2.32.0.0/14", ...]
    }
  ]
}
```

### `proxy-vpn-ips.json`
デフォルトで許可するプロキシ・VPNサービスのIPアドレスを管理します。

```json
{
  "name": "許可済みプロキシ・VPN IPリスト",
  "description": "デフォルトで許可するプロキシ・VPNサービスのIPアドレス",
  "version": "1.0.0",
  "lastUpdated": "2026-03-27",
  "services": {
    "CroxyProxy": {
      "name": "CroxyProxy",
      "type": "Proxy",
      "ips": ["64.227.120.231", "157.230.79.247", ...]
    }
  }
}
```

## 🔄 データ更新方法

### 1. 手動更新
JSONファイルを直接編集して更新します。

### 2. サーバー再読み込み
管理者ページの「データ再読み込み」ボタンを使用して、サーバーを再起動せずにデータを更新できます。

### 3. APIによる更新
```bash
# データ再読み込み
curl -X POST http://localhost:5000/api/geoip/reload-data

# データ情報取得
curl http://localhost:5000/api/geoip/data-info
```

## 🔧 メンテナンス

### 定期的な更新
- **推奨頻度**: 月1回
- **更新対象**: 新規IP範囲、サービス変更
- **確認項目**: JPNIC割り当て、プロキシサービスIP

### データソース
- **日本IP**: JPNIC公式割り当て情報
- **プロキシ・VPN**: 各サービス公式情報
- **検証**: 実際の接続テスト

### エラー処理
- ファイルが存在しない場合：フォールバックデータを使用
- JSON形式エラー：サーバーログに記録
- 読み込み失敗：基本的なIP範囲で動作継続

## 🛠️ 開発者向け

### データ形式の要件
- **エンコーディング**: UTF-8
- **形式**: JSON
- **検証**: サーバー起動時に構文検証

### 新規プロバイダー追加
```json
{
  "provider": "新しいプロバイダー名",
  "ranges": ["xxx.xxx.xxx.xxx/xx", ...]
}
```

### 新規サービス追加
```json
{
  "サービス名": {
    "name": "表示名",
    "type": "Proxy|VPN",
    "ips": ["xxx.xxx.xxx.xxx", ...]
  }
}
```
